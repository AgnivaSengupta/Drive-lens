# 🗂️ Drive Lens: Conversational Google Drive Agent

Drive Lens is a specialized AI agent that allows you to search and interact with your Google Drive using natural language. It combines a **FastAPI** backend powered by **LangGraph** for sophisticated tool-calling with a clean **Streamlit** frontend.

## Key Features

- **Natural Language Search**: Find files using conversational queries (e.g., "Find all PDFs modified last week about 'budget'").
- **Multi-Session Memory**: The agent remembers your name and preferences across different chat sessions using a local SQLite-based long-term memory system.
- **Persistence**: Chat history and user sessions are persisted in SQLite, surviving backend restarts.
- **Real-time Tool Output**: View the exact Drive API query generated by the LLM and the raw search results.
- **Profile Management**: Switch between demo users or create custom profiles with unique memory banks.

## Setup & Installation

### 1. Prerequisites

- Python 3.10+
- A Google Cloud Service Account JSON key (placed in `credentials/service-account.json`).
- A Google Drive Folder ID to scope searches.

### 2. Environment Setup

```bash
# Create and activate virtual environment
python -m venv venv
source venv/bin/activate  # On Windows: venv\Scripts\activate

# Install dependencies
pip install -r requirements.txt
```

### 3. Configuration

Create a `.env` file in the project root:

```env
# Google Drive
FOLDER_ID=your_google_drive_folder_id
SERVICE_ACCOUNT_FILE=./credentials/service-account.json

# Backend
BACKEND_URL=http://localhost:8000

# LLM Selection (Choose one: gemini, openai, groq, or openrouter)
LLM_PROVIDER=gemini
MODEL_NAME=gemini-2.0-flash

# Required based on your LLM_PROVIDER
GOOGLE_API_KEY=your_api_key_here
OPENAI_API_KEY=your_api_key_here
GROQ_API_KEY=your_api_key_here
```

## Running the Application

1. **Start the Backend Server**:

   ```bash
   uvicorn backend.main:app --reload
   ```

   The backend provides the API for chat, session management, and memory extraction.

2. **Start the Streamlit Frontend**:
   ```bash
   streamlit run frontend/app.py
   ```
   The UI will be available at `http://localhost:8501`.

## Architecture

- **Backend (`/backend`)**:
  - `main.py`: FastAPI routes for chat, sessions, and history.
  - `agent/`: LangGraph orchestration, tool definitions, and LLM configuration.
  - `cache/`: Caching mechanism for Drive API responses.
  - `data/`: Local SQLite storage for sessions, checkpoints, and long-term memory.
  - `drive/`: Logic for interacting with the Google Drive API.
  - `middleware/`: FastAPI middleware components.
  - `models/`: Pydantic data models for API requests and responses.
- **Frontend (`/frontend`)**:
  - `app.py`: Streamlit interface with custom CSS styling and session management.
  - `components/`: Reusable UI components like file cards.

## Persistence Details

- `sessions.db`: Stores user IDs and chat session metadata.
- `checkpoints.db`: Stores the actual LangGraph conversation state (thread history).
- `long_term_memory.db`: Stores extracted facts and preferences for every user.
